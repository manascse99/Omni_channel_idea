1. Database Overview 

OmniBank AI uses MongoDB as its primary database. All customer data, conversations, 
messages, agents, teams, and OTP tokens are stored across 6 collections. Every collection is 
connected via ObjectId references — no data is duplicated across collections. 

Design Rule: Build the database schema first, then write backend APIs around it. Your API routes 
should simply be thin wrappers that read and write to these collections.

1.2 Collection Relationships 

Here is how the 6 collections relate to each other: 

users (1) 
  └── conversations (1)        → conversations.userId = users._id 
        └── messages (many)    → messages.conversationId = conversations._id 
                               → messages.userId = users._id 

agents (many) 
  └── teams (1)                → agents.teamId = teams._id 
        └── conversations      → conversations.assignedTo = agents._id 
                               → conversations.assignedTeam = teams._id 

otpTokens                      → otpTokens.agentId = agents._id 
  └── TTL index auto-deletes after expiry

2.2 conversations Collection 

One document per user. This is the master record for a customer's entire interaction history with 
the bank, across all channels. 

{ 
  _id:          ObjectId, 
  userId:       ObjectId,              // ref: users._id (1:1 relationship) 
  status:       String,                // 'open'|'ai-handling'|'pending-
review'|'escalated'|'resolved' 
  assignedTo:   ObjectId | null,       // ref: agents._id 
  assignedTeam: ObjectId | null,       // ref: teams._id 
  intent:       String | null,         // top AI intent: 'loan_inquiry', 
'grievance' etc. 
  sentiment:    String | null,         // 'positive'|'neutral'|'negative' 
  lastMessage:  String,               // preview text for conversation list 
  lastChannel:  String,               // most recent channel used 
  aiSummary:    String | null,         // rolling AI context summary (2-3 
sentences) 
  unreadCount:  Number,               // unread messages for agent badge 
  createdAt:    Date, 
  updatedAt:    Date                  // updated on every new message 
}

2.3 messages Collection 

Every individual message — from customer, agent, or AI — is stored as a separate document. 
This is the highest volume collection in the system. 

{ 
  _id:            ObjectId, 
  conversationId: ObjectId,            // ref: conversations._id 
  userId:         ObjectId,            // ref: users._id (the customer) 
  channel:        String,              // 'whatsapp'|'email'|'webchat' 
  senderType:     String,              // 'user'|'agent'|'ai' 
  content:        String,              // message text / email body 
  isRead:         Boolean,             // false = unread, true = seen by agent 
  timestamp:      Date,               // when message was sent/received 
  metadata: { 
    whatsappMsgId:  String | null,     // Meta message ID (deduplication) 
    emailSubject:   String | null,     // email subject line 
    emailMessageId: String | null,     // email Message-ID header 
    agentId:        ObjectId | null,   // if senderType = 'agent' 
  } 
}

•

{ conversationId: 1, timestamp: 1 } — compound index for fetching full thread in order 
{ conversationId: 1, isRead: 1 } — for fast unread count per conversation 
{ 'metadata.whatsappMsgId': 1 } — sparse, for WhatsApp deduplication check 
{ timestamp: -1 } — for recent messages queries across all conversations 

Never query messages without conversationId in the filter. Without it, MongoDB does a full collection 
scan. Always query like: messages.find({ conversationId: id }).sort({ timestamp: 1 })

2.6 otpTokens Collection 

Stores short-lived OTP codes for agent login. MongoDB's TTL index automatically deletes 
expired tokens — no cron job needed.

Indexes — otpTokens 

• 
• 
• 

{ agentId: 1 } — for quick OTP lookup on verify 
{ createdAt: 1 } — TTL index: expireAfterSeconds: 300  (auto-deletes after 5 min) 
{ phone: 1 } — for rate limiting (max 3 OTPs per phone per 10 min) 

TTL Index Magic: When you create { createdAt: 1 } with expireAfterSeconds: 300, MongoDB 
automatically deletes the OTP document after 5 minutes. No cron job, no manual cleanup — 
MongoDB handles it internally.

3. Setup & Connection 

3.1 Install Mongoose 
npm install mongoose 

3.2 MongoDB Atlas Setup Steps 

1.  Go to mongodb.com/atlas → Create free account 
2.  Create a free M0 cluster (512MB free — enough for MVP) 
3.  Create a database user (username + password) 
4.  Whitelist IP: 0.0.0.0/0 for development (restrict in production) 
5.  Copy the connection string from Atlas → Drivers → Node.js 

3.3 Connection Code (db.js) 

// src/config/db.js 
const mongoose = require('mongoose'); 

const connectDB = async () => { 
  try { 
    await mongoose.connect(process.env.MONGODB_URI, { 
      dbName: 'omnibank' 
    }); 
    console.log('MongoDB connected'); 
  } catch (error) { 
    console.error('MongoDB connection failed:', error.message); 
    process.exit(1); 
  } 
}; 

module.exports = connectDB;

# .env 
MONGODB_URI=mongodb+srv://<username>:<password>@cluster0.xxxxx.mongodb.net/omnibank 

3.5 Creating All Indexes (Run Once) 

Run this script once after connecting to create all required indexes: 

// scripts/createIndexes.js 
const mongoose = require('mongoose'); 
require('dotenv').config(); 

async function createIndexes() { 
  await mongoose.connect(process.env.MONGODB_URI, { dbName: 'omnibank' }); 
  const db = mongoose.connection.db; 

  // users

  await db.collection('users').createIndex({ phone: 1 }, { unique: true, sparse:

  await db.collection('users').createIndex({ email: 1 }, { unique: true, sparse:

  await db.collection('users').createIndex({ lastSeen: -1 });

  await db.collection('conversations').createIndex({ status: 1 }); 
  await db.collection('conversations').createIndex({ updatedAt: -1 }); 
  await db.collection('conversations').createIndex({ assignedTo: 1 }); 
  await db.collection('conversations').createIndex({ assignedTeam: 1 }); 

  // messages 
  await db.collection('messages').createIndex({ conversationId: 1, timestamp: 1 }); 
  await db.collection('messages').createIndex({ conversationId: 1, isRead: 1 }); 
  await db.collection('messages').createIndex( 
    { 'metadata.whatsappMsgId': 1 }, { sparse: true } 
  ); 

  // agents 
  await db.collection('agents').createIndex({ phone: 1 }, { unique: true }); 
  await db.collection('agents').createIndex({ teamId: 1 }); 
  await db.collection('agents').createIndex({ status: 1 }); 

  // otpTokens — TTL index (auto-deletes after 5 minutes) 
  await db.collection('otpTokens').createIndex( 
    { createdAt: 1 }, { expireAfterSeconds: 300 } 
  ); 
  await db.collection('otpTokens').createIndex({ phone: 1 }); 

  console.log('All indexes created successfully'); 
  process.exit(0); 
} 

createIndexes();
